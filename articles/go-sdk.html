The Official Go SDK

Currently, the tools in the official Go SDK are the most used tools to develop Go projects. In Go 101 article series, all examples are compiled and verified with the official Go compiler.

This article will introduce how to setup the Go development environment and how to run simple Go programs. Some tools provided in the official Go SDK will also be introduced.

Install Go SDK

Please download the official Go SDK and install it according the instructions at the same page.

The version of an official Go SDK release is consistent with the highest Go language verison the release supports. For example, Go SDK 1.9.x supports all Go versions from 1.0 to Go 1.9.

The path to the bin subfolder in the Go SDK installation root path must be put in the PATH environment variable to execute the tools in the SDK.

(Before Go SDK 1.9, a GOROOT environment variable must be configured as the root path of the SDK installation. Since Go SDK 1.9, this environment variable becomes optional and is recommended not to set it.)

Before Go SDK 1.8, a GOPATH environment variable must be set. Since Go SDK 1.8, the default value of this environment variable is the path of the go folder under the home directory of the current user. If you don't like the default GOPATH, you still can specify it manually. The GOPATH environment variable can contain multiple paths.

The bin subfolder in the first path specified in the GOPATH environment variable is highly recommended to be put in the PATH environment variable to run some third-party Go programs. The go install command will put the result executables in the bin folder (below will introduce all kinds of go sub-commands).

Up to Go 1.9, the official Go SDK recommends all of your Go packages are put in the src subfolder in any path specified in the GOPATH environment variable, especially when a Go project depends on some third party packages. However, all examples in Go 101 series are very simple so that you can put the example files in any folder. Packages will be introduced in the packages and imports article.

The Simplest Go Program

Let's write a simple exmaple and learn how to run simple Go programs.

The following program is the simplest Go program. 
package main

func main() {
}

The words package and func are two keywords. The two main words are two identifiers. Keywords and identifiers are introduced in the next article.

The first line package main specifies the package name (main here) of the soruce file.

The second line is a blank line, to make the code get a better readability.

The remaining code declares a function which is also called main. We can think the main function in the main package of a program specifies the entry point of the program, though there would be some other user code getting executed before the main function in Go.

Run Go Programs

The official Go SDK requires that every Go source code file must ends with a .go extension. Here, we assume the above source code saved in a file named simplest-go-program.go.

Open a terminal and change the current directory to the directory which contains the above source file, then run 

$ go run simplest-go-program.go

Nothing is output? Yes, this program does nothing.

If there are some syntax errors in the source code, then there would be some compile errors reported.

Mote. if there are multiple source files in the main package of a program, then we should run the program with the following command 

$ go run *.go

Note, the go run command is not recommended to compile and run large Go projects. It is just a convenient way to run simple Go programs, just like the ones in the Go 101 article series. For large Go projects, the go build and go install commands should be used instead.

More go Subcommands

The three commands, go run, go build and go install, will only show code syntax errors if they fail. They don't show code warnings (a.k.a, possible code logic errors). We can use go vet command to check and report such warnings.

We can use the go fmt command to format Go source code with a consistent coding style.

We can use the go get command to clone and checkout a remote third-party go package repository. go get requires the corresponding version control tool must be installed. If the command successes, the third-party go package will be put in the src folder under the first path specified in the GOPATH environment variable.

The go test command is used to run tests and benchmarks.

We can use the go doc command to view Go documentations.

You can run go command without any arguments to show what subcommands are supported.

The Go 101 article series will not explain much more on how to use the tools provided by the official Go SDK. Please read the official documentations for details.

View Go Documentations

We can view all kinds of Go documentations at the official Go website golang.org.

We can also run godoc -http=:9999 to start a local clone of the official website at localhost:9999.

Please notes, currently (v1.9), if the GOROOT environment variable is not configured, we must specify the goroot flag when running the local godoc server. For example, godoc -http=:9999 -goroot path/to/go/sdk.
