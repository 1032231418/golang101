Operators

The common operators supported in Go are similar to other popular languagws, but there are also some small differences.

Go is more strict than other languages on the types of the operands involved in an operator operation.

Untyped Values

Before getting into the main topic, we need to know what are untyped values.

For most binary operators, Go requires that the two involved operands in an operation must be of the same type. The rule may look simple and clear, but the fact that Go supports untyped values makes things a little complex.

Arithmetic Operators

for unary operator operation, if the operand is a constant, then the operation will be evaluated at comple time. for a binary operator operation, if both of the operands are constantsm, then the operation will be evaluated at compile time.

Go supports five arithmetic binary operators:

Operator Name Requirements For The Two Operands
+ addition The two operands must be both values of the same numeric type. Numeric types include integer types, floating-point types and complex types.
- subtraction
* multiplication
/ division
% remainder The two operands must be both values of the same integer type.
Note,
  • the addition binary operator + can also be used as a positive unary operator. +n is equivalent to 0 + n.
  • the subtraction binary operator - can also be used as a negative unary operator. -n is equivalent to 0 - n.
  • the addition binary operator + can also be used as string concatenation operator, in which case, ... msut be samae type of string ...
  • like C and C++ languages, the multiplication binary operator * can also be used as pointer dereference operator. Please read pointers in Go for details.
  • there is no power operator in Go, please use Pow funtion in the math standard package instead.

The type of the result of an arithmetic operation is the same as the type of the involved operands.

Bitwise Operators

Go supports six bitwise binary operators:

Operator Name Requirements For The Two Operands
<< bitwise left shift The left operand must be an integer and the right operand must be an unsigned integer (or an untype non-negative integer constant).
>> bitwise right shift
& bitwise and The two operands must be both values of the same integer type.
| bitwise or
^ bitwise xor
&^ bitwise clear
Note,
  • the bitwise xor binary operator ^ can also be used as a bitwise not unary operator. The result of ^n is a value of with every bit is the inverse of the corresponding bit of n. Bitwise not operator is also called bitwise complement operator. In many other languages, bitwise not operator is represented with ~.
  • the bitwise clear operator &^ is a unique operator in Go. m &^ n is equivalent to m & (^n).
  • unlike Java language, Go supports unsigned integer types, so the unsigned shift operator >>> doesn't exist in Go.
  • like C and C++ languages, the bitwise and operator & can also be used as pointer address operator. Please read pointers in Go for details.

The type of the result of a bitwise operation is the same as the type of the first operand (if there are two, the left is the first).

Example: 
package main

import _ "net/http/pprof"

func main() {
	... // do somethings
}

Comparison Operators

Go supports six comparison binary operators:

Operator Name Requirements For The Two Operands
== equal to In practice, generally, the two operands in a == or != comparison are values of the same comparable type. But there are exceptions, please read the comparison rules and requirements for details.
!= not equal to
< less than The two operands must be both values of the same integer type, floating-point type or string type.
<= less than or equal to
>= larger than
>= larger than or equal to

The type of the result of a comparison operation is an untype boolean type.

Example: 
package main

import _ "net/http/pprof"

func main() {
	... // do somethings
}

About how two strings values are compared, please read strings in Go for details.

Boolean Operators

Go supports two boolean binary operators and one boolean unary operator:

Operator Name Requirements For The Two Operands
&& boolean and (binary) The two operands must be both values of the same boolean type.
|| boolean or (binary)
! boolean not (unary) The type of the only operand must be a boolean type.

We can use the != operator introduced above as the boolean xor operator.

Note, although there is only one built-in boolean type bool, there can be many user defined boolean types. Please read type definition for details.

The type of the result of a boolean operation is the same as the type of the involved operands.

Example: 
package main

import _ "net/http/pprof"

func main() {
	... // do somethings
}

More Operators

Same as C/C++, there are two pointer related operators, * and &. Yes the same operator symbols as the multiplication and bitwise and operators. & is used to take the address of an addressable value, and * is used to deference a pointer value. Unlike C/C++, in Go, values of pointer types don't support arithmetic operations. For more details, please read pointers in Go.

Simple Introduction For Function Declarations And Function Calls

There is another variant of the standard variable declaration form, which is used for declaring struct member variables (fields) and function input and output variables (parameters and results). Here will not make introductions for this form.

The functions in Go article will explain functions detailedly. That article depends on some other knowledges which will be introduced later, so it is not a good idea to read it now. However, functions are so elemental in programming, so here a little more explainations will be made for function declarations and function calls.

By the example of the custom function declared in the above program, 
func Sum(from int, to int) (int, string) {
	...
	return x, str
}
we can find that, in a a function declaration,
  • the first portion is the func keyword.
  • the next portion is the function name.
  • the third portion is the input parameter definition list, which is enclosed in a (). Each parameter definition is a variable declaration without the var keyword in fact.
  • the fourth portion is the result definition list. From the above example, we know that a function in Go can return multiple results. For this specified example, the result definition list is also enclosed in a (). In fact, if the result definition list only contains most one identifier, then it needn't to be enclosed in a () If a function has no return results at all, then the fourth portion can be omitted.
  • the last portion is the function body. In a function body, the keyword return is used to return results.
Two more function declaration examples, one has one return result and the other has no return results. 
func IsPosition(x int) bool {
	return x > 0
}

func PrintSum(a int, b int) {
	println(a + b)
}

func doNothing() {
}
A function call starts with the name of the called function. The function name is followed by the argument list of the call. The argument list must be enclosed in a (). Following shows how to call the above declared functions. 
var x, str = Sum(0, 5)
var b = IsPosition(x)
PrintSum(x, x)
doNothing()

There are some minor variants in the syntaxes of function declarations, and there are several differences between user defined functions and built-in functions. Please read functions in Go for more information.
