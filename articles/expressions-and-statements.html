Expressions, Statements And Simple Statements

This article will introduce expressions and statements in Go.

Simply speaking, expressions represent values and statements represent operations. However, in fact, some special expressions each may represent several values, and some statements may be composed of several sub operations/statements. And some expressions can also be used as statements.

Simple statements are some special statements. In Go, some portions in all kinds of control flows must be simple statements, and some portions must be expressions. Control flows will be introduced in the next Go 101 article.

This article will not make accurate definitions for expressions and statements. It is hard to achieve this. This article will only list some express and statement cases. Not all kinds of expressions and statements will be covered in this article, but all kinds of simple statements will be listed.

Some Expression Cases

A value literal, a named constant and a variable are all expressions. In fact, later we will learn that custom declared functions are function values, so they are also expressions.

The operations (without the assignment part) using the operators introduced in the last article are all expressions. There are more operator operations which will be introduced in other articles are also expressions.

By the way, if all the operands involved in an operator operation (without the assignment part) are constants, then the operator operation is called a constant expression. Constant expressions are evaluated at compile time instead of run time.

If a function returns exact one result, then its calls (without the assignment parts) are expressions. Calls of functions without results are not expressions. Calls of functions with more than one results can are expressions, but such expressions can used be used at some specific senarios. Go functions will explained detailedly in the article functions in Go later.

Methods can be viewed as special functions. So the above mentioned function cases also apply to methods. Methods will explained detailedly in the article method in Go later.

Channel receive operations (without the assignment parts) are also expressions. Channel operations will be explained in the article goroutine and channel introduction later.

Some Simple Statement Cases

There are six kinds of simple statements.
  1. short variable declaration forms
  2. pure value assignments (not mixing with variable declarations)
  3. function/method calls and channel receive operations without the assignment parts.
  4. channel send operations.
  5. nothing. This is called blank statement.
  6. x++ and x--.

Again. channel receive and sent operations will be incroduced in the article goroutine and channel introduction.

Yes, Go supports the two syntaxes. However, the two forms can't be used as values. In other words, they are not expressions. x++ is equivalent to x = x + 1, and x-- is equivalent to x = x - 1. Go doesn't support the ++x and --x syntaxes.

Some Non-Simple Statement Cases

The standard variable declaration forms are statements but not simple statements.

Constant, type and function declarations are non-simple statements. A function declaration statement may contains many sub statements.

All the control flows introduced in the next article basic control flows are also non-simple statements.
